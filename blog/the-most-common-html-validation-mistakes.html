The most common HTML validation mistakes

guide
30 oct, 2022

Have you ever validate HTML markups? I do. One time I had interesting task: to validate a whole project to improve SEO things. I came to this project at the very end when almost everything was done by other developers. There was a lot of work as it turned out after doing a quick HTML validation check!

In this article I have compiled below a comprehensive list of the most common HTML validation mistakes developers often do. Let's go through them one by one.

But before we start...

Conventions used in this guide

There are two types of validation mistakes:

  1. warning

    Not critical mistake that may or may not cause problems for users, perhaps depending on the user's browser, but that you should know about. These may include mistakes against HTML standard but that are less likely to be problematic.

  2. Error

    Critical mistake that may cause problems for users. Errors can also be issues that are flagrantly against HTML standards.

For convenience, next to each HTML mistake below will be its type: error or warning. Also each mistake provides with examples.

There are a few types of HTML tags. First type is paired. Any paired tag consists of an opening tag and a closing tag as its companion tag. Most of HTML tags are paired. For example, the most famouse HTML tag <div> is paired. In contrast to paired tags, unpaired tags have only an opening tag and does not have a closing tag or a companion tag. For example, HTML horizontal line tag <hr> is unpaired or self-closing.

A unclosed tag is enough to drive anybody nuts. It's very difficult to track especially when a web page is long and complex. Nowadays with powerful source-code editors and with a rich set of tools such as unpaired tags finders is not a big problem. In any case, always make sure every paired tag has the opening tag and the closing tag.

WarningSection lacks heading

Accroding to HTML5 spec, the theme of each section should be identified, typically by including a heading <h1>-<h6> element as a child of the section element.

You have three solutions here:

  1. Ignore the warning because this message is a non-normative usage recommendation
  2. Replace <section> element to <div>
  3. Add a heading <h1>-<h6> tag to your section element
