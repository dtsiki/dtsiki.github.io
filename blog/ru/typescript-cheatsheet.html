Шпаргалка по TypeScript: всё важное на одной странице

12 july 2026

Большая шпаргалка по синтаксису и основным возможностям TypeScript. Бесплатно, без регистрации и смс.

Содержание

Установка TypeScript

Перед установкой TypeScript проверьте, что у вас стоит Node.js. Если есть сомнение — откройте терминал (командную строку) и напишите:

node --version
npm --version

Если появились цифры типа v18.18.0 и 9.8. — всё ок. Если нет, то идём на nodejs.org, качаем LTS-версию, устанавливаем. Это всё бесплатно.

Есть несколько вариантов установки TypeScript: глобальная, локальная вручную и автоматическая через сборщики.

Глобальная установка

Позволяет использовать команды компилятора TypeScript в любом месте на вашем компьютере. Удобно для экспериментов и небольших скриптов.

npm install -g typescript

Чтобы проверить, что TypeScript установился, просто введите команду проверки текущий версии:

tsc -v

Затем вы просто создаёте файлы с расширением .ts, в котором пишите TypeScript-код, сохраняете изменения, а для компиляции этих файлов запускаете команду:

tsc <название-файла>.ts

После компиляции в директории проекта будет создан файл <название-файла>.js, который будет содержать JavaScript-код, скомпилированный из вашего TypeScript-кода.

Если нужно скомпилировать весь проект используйте команду:

npx tsc

Локальная установка вручную для проекта

Подойдёт для разработки конкретного приложения.

В терминале перейдите в папку проекта и выполните команду:

npm install --save-dev typescript

В зависимостях проекта появится TypeScript, а в директории проекта появится файл tsconfig.json.

Автоматическая установка через сборщики

Многие современные фреймворки и сборщики, например Vite или Next.js, настраивают TypeScript самостоятельно при создании проекта. Вам просто нужно выбрать соответствующий пункт при инициализации проекта.

Если вы создаёте проект с помощью утилиты create-react-app, то достаточно указать флаг --template typescript:

npx create-react-app <название-репозитория>  --template typescript

Будет создан шаблон React-приложения с файлом tsconfig.json, TypeScript в зависимостях и с файлами с расширением .ts/.tsx.

Create React App официально признан устаревшим (deprecated) инструментом для создания React-приложений с 14 февраля 2025 года. Инструмент больше не рекомендуется для создания новых проектов из-за медленной сборки на Webpack и отсутствия поддержки современных возможностей React 19. Рекомендуется переходить на более современный методы, например, Vite.

Песочница TypeScript

У TypeScript есть официальная онлайн-песочница, где можно писать, тестировать и компилировать код TypeScript в браузере без установки пакетов. Там можно делать всё что душа захочет и сразу же видеть результат компиляции в JavaScript-код.

Типы данных и их типы

Аннотация типов(Type Annotations)

Это способ явно указать, какие данные ожидаются в переменных, параметрах функций или возвращаемых значениях. В TypeScript она задается с помощью двоеточия : и названия типа, который идет сразу после имени переменной или функции:

1let isAwesome: boolean = true;
2let answer: number = 42;
3let username: string = "@dtsiki";
4let notDefined: undefined = undefined;
5let isNull: null = null;

В скомпилированном JavaScript аннотации удаляются, оставляя код чистым JavaScript:

bundle.js
1let isAwesome = true;
2let answer = 42;
3let username = "@dtsiki";
4let notDefined = undefined;
5let isNull = null;

Выводимость типов(Type Inference)

TypeScript может сам автоматически определять типы переменных и такую аннотацию можно не прописывать:

1let isAwesome = true;
2let answer = 42;
3let username = "@dtsiki";
4let notDefined = undefined;
5let isNull = null;

Это способность TypeScript автоматически определять тип переменной называется выводимость типов. Также TypeScript справляется сам и может определять тип возвращаемого значения из функции, типы у массивов с явными значениями, тип колбэков у методов массивов, в остальных случаях лучше помогать ему и прописывать аннотации.

Приведение типов(Type Assertions)

Это способ, позволяющий явно указать компилятору точный тип переменной, когда он не может определить его самостоятельно:

1let someValue: any = "Привет, TypeScript";
2let strLength: number = (someValue as string).length;

Важно понимать: это не преобразование, а лишь подсказка компилятору.

При сборке в JavaScript эти конструкции будут удалены.

Примитивные типы данных

В TypeScript переехали из JavaScript все примитивные типы данных: number, string, boolean, null, undefined, bigint, symbol

1let isAwesome: boolean = true;
2let answer: number = 42;
3let username: string = "@dtsiki";
4let notDefined: undefined = undefined;
5let isNull: null = null;
6let id: symbol = Symbol("id");
7let REALLYBIGNUMBER: bigint = 123n;

TypeScript их не меняет, а просто добавляет к ним соответствующие аннотации типов.

any, unknown, never, void

В TypeScript появились новые типы данных: any, unknown, never, void. Этих типов нет в JavaScript. Они существуют только до компиляции, а после того как TypeScript компилируется в JavaScript, они исчезают.

any

Тип данных, который представляет любое значение:

1let anyVar: any = 42;
2anyVar = "Yay";
3anyVar = true;
4
5const doSomething = (data: any) => {
6  console.log(data);
7};
8
9doSomething(42);
10doSomething("Привет");
11doSomething({ name: "Кеша", age: 4 });

Хотя кажется, что использование any даёт гибкость, лучше избегать его, поскольку это может снизить надежность и читаемость кода. Лучше использовать явные типы данных.

unknown

Безопасная версия any: тоже можно присвоить что угодно, но перед использованием тип нужно проверить:

1let input: unknown = 'Что-то';
2
3// input.toUpperCase(); // Ошибка! TS не уверен
4
5if (typeof input === 'string') {
6  console.log(input.toUpperCase()); // Теперь можно
7}

Может показаться, что unknown работает так же, как any, но между ними есть различие. Тип any по сути отключает проверку типов и позволяет выполнять любые операции, например, обращаться к свойствам переменной. Тип unknown запрещает это и требует предварительной проверки типа переменной, либо приведения к нужному типу.

never

Используется для обозначения значений, которые никогда не происходят. Можно использовать для функций, которые выбрасывают ошибку или для бесконечных циклов:

1function throwError(message: string): never {
2  throw new Error(message);
3}
4
5function infiniteLoop(): never {
6  while (true) {}
7}

void

Обозначает отсутствие значения. Обычно используется для функций, которые ничего не возвращают:

1function logMessage(message: string): void {
2  console.log(message);
3}

Массивы

Массивы в TypeScript работают так же как и в JavaScript. В TypeScript есть два способа протипизировать массивы:

  1. С помощью синтаксиса T[]:

    1let abc: string[] = ['a', 'b', 'c'];
2let nums: number[] = [1, 2, 3, 4, 5];
3let flags: boolean[] = [true, false, true];

  2. С помощью синтаксиса Array<T> (см. Дженерики):

    1let abc: Array<string> = ['a', 'b', 'c'];
2let nums: Array<number> =  [1, 2, 3, 4, 5];
3let flags: Array<boolean> = [true, false, true]

Оба варианта равнозначны, но чаще используют первый с T[].

При этом массивы являются строго типизированными — если изначально массив содержит строки, то в будущем он сможет работать только со строками. Если попытаться добавить в такой массив число, то будет получена ошибка:

1const abc: string[] = ['a', 'b', 'c'];
2abc.push(42); // Error: Argument of type 'number' is not assignable to parameter of type 'string'

Кортежи (Tuples)

Кортежи — это специальные массивы с фиксированной длиной и строго определенными типами для каждого элемента, даже если они разные. В отличие от обычных массивов, порядок элементов в кортеже имеет значение:

1let user: [number, string] = [42, "Daria"];

После компиляции в JavaScript кортежи превращаются в обычные массивы.

Объекты

Как и в JavaScript, в TypeScript можно создавать сложные объекты, которые состоят из других объектов, в том числе массивов и примитивных данных. Объекты тоже типизируются в TypeScript: c помощью определения структуры явно описывается какие свойства и типы значений должен содержать объект:

1let user: { id: number; name: string; isAwesome: boolean } = {
2    id: 42,
3    name: "dtsiki",
4    isAwesome: true
5};

TypeScript позволяет делать какие-то свойства необязательными. Для этого после названия свойства указывается знак вопроса ?:

1let user: { id: number; name: string; isAwesome: boolean, age?: number };

Такое определение структуры объектов в TypeScript используется редко. Для определения структуры объектов следует использовать интерфейсы или типы.

Функции

Типизация входных параметров

В TypeScript можно типизировать аргументы функций:

1function sum(a: number, b: number) {
2  return a + b;
3}

Теперь когда аргументы протипизированы, TypeScript будет проверять их при вызове функции:

1sum(2, 2);
2sum("Hello", 42); // Error: Argument of type 'string' is not assignable to parameter of type 'number'
3sum('2', '2'); // Error: Argument of type 'string' is not assignable to parameter of type 'number'

Стрелочные функции тоже типизируются:

1const sayHello = (name: string) => {
2  return `Привет, ${name}`;
3};

Можно создать тип для функции, чтобы использовать его повторно в переменных или параметрах других функций:

1type MathOperation = (a: number, b: number) => number;
2
3const sum: MathOperation = (x, y) => x + y;
4const multiply: MathOperation = (x, y) => x * y;

Аргументы функций могут быть необязательными. Для того, чтобы сделать параметр необязательным, после его названия указывается знак вопроса ?. В примере ниже таким образом аргумент name может принимать значение как string, так и undefined:

1function sayHello(name?: string) {
2  return `Привет, ${name ?? "таинственный гость"}`;
3}

Параметрам можно задать значение по умолчанию:

1function sayHello(name: string = "таинственный гость") {
2  return `Привет, ${name}`;
3}

Важно помнить, что необязательные параметры должны идти после обязательных.

Типизация возвращаемого значения

Также можно типизировать возвращаемое из функции значение:

1function getFavoriteNumber(): number {
2  return 42;
3}

Если функция ничего не возвращает, то для этого используем специальный тип void:

1const log = (message: string): void => {
2  console.log(message);
3};

TypeScript определяет тип возвращаемого значения функции на основе оператора return, поэтому писать тип возвращаемого значения необязательно.

Перегрузки функций (Function Overloads)

Можно заставить функцию вести себя по-разному в зависимости от аргументов с помощью перезагрузки:

1function format(value: string): string;
2function format(value: number): string;
3
4function format(value: string | number): string {
5  if (typeof value === "number") {
6    return value.toFixed(2);
7  }
8  return value.trim();
9}

Типы (Types)

С помощью типов можно описывать любые структуры данных: примитивные типы, объекты, функций, массивов и т.д. Ключевое слово здесь — любые.

Не следует путать с типами данных.

Типы записываются с помощью ключевого слова type.

1type User = {
2  id: number;
3  name: string;
4  isAwesome: boolean;
5  age?: number;
6};
7
8const user: User = { id: 42, name: 'dtsiki', isAwesome: boolean };

Можно описывать не только объекты, можно и примитивы:

1type ID = string | number;
2const stringId: ID = 'asdfgh';
3const numberId: ID = 42;
4const bigintId: ID = 123n; // Ошибка: Type 'bigint' is not assignable to type 'ID'

Функции:

1type MathOperation = (a: number, b: number) => number;
2
3const sum: MathOperation = (x, y) => x + y;
4const subtract: MathOperation = (x, y) => x - y;
5const concat: MathOperation = (x, y) => `${x}${y}`; // Ошибка: Type 'string' is not assignable to type 'number'

Массивы:

1type StringArray = string[];
2type NumberArray = Array<number>;
3type MixedArray = (string | number)[];
4
5const abc: StringArray = ['a', 'b', 'c'];
6const nums: NumberArray = [1, 2, 3];
7const mixed: MixedArray = ['hello', 42];

Кортежи:

1type Point = [number, number];

Типы в TypeScript можно называть с префиксом T, например, TUser вместо User. Это не обязательное правило TypeScript, а соглашение, пришедшее из других языков для улучшения читаемости кода. Это помогает мгновенно отличить тип от других значений, например интерфейсов, классов или объектов.

Объединение и пересечение типов

Для создания нового типа можно комбинировать несколько типов с помощью объединения (Union) и пересечения (Intersection).

Для объединения используется символ |. В результате объединения получается новый тип, который будет содержать всё, что есть в типах, участвующих в объединении:

1type Status = "loading" | "success" | "error";

Для пересечения используется символ &. Новый тип будет содержать свойства всех указанных при пересечении типов:

1type A = { a: number };
2type B = { b: string };
3
4type C = A & B;

Также с помощью пересечения можно расширять типы:

1type User = { name: string };
2type Admin = User & { role: string };

Ограничения типов

  • Типы нельзя переопределять:

    1type User = { name: string };
2type User = { age: number }; //  Error: Duplicate identifier 'User'

Интерфейсы(Interfaces)

Интерфейсы подходят для описания структуры объектов, функций и классов. Ключевое здесь — в первую очередь для объектов.

Интерфейсы записываются с помощью ключевого слова interface:

1interface User {
2  id: number;
3  name: string;
4  isAwesome: boolean;
5  age?: number;
6}
7
8const user: User = { id: 42, name: 'dtsiki', isAwesome: true };

Интерфейсы в TypeScript можно называть с префиксом I, например, IUser вместо User. Это не обязательное правило TypeScript, а соглашение, пришедшее из других языков для улучшения читаемости кода. Это помогает мгновенно отличить интерфейсы от других значений, например типов, классов или объектов.

Интерфейсы могут описывать функции:

1interface MathOperation {
2  (a: number, b: number): number;
3}

Объединение интерфейсов(Declaration Merging)

Интерфейсы могут быть объединены, но если они имеют одно и то же имя. TypeScript автоматически сам слияет интерфейсы. Это позволяет добавлять новые свойства к существующим интерфейсам:

1interface User {
2  name: string;
3}
4
5interface User {
6  isAwesome: boolean;
7}
8
9const user: User = {
10  name: "dtsiki",
11  isAwesome: true,
12};

Расширение интерфейсов

Интерфейсы могут расширять другие интерфейсы с помощью ключевого слова extends:

1type Department = 'frontend' | 'backend';
2
3interface User {
4  name: string;
5  isAwesome: boolean;
6}
7
8interface Developer extends User {
9  department: Department;
10}

Можно расширять несколько интерфейсов сразу:

1type Department = 'frontend' | 'backend';
2
3interface User {
4  name: string;
5  isAwesome: boolean;
6}
7
8interface Developer extends User {
9  department: Department;
10}
11
12interface TeamLead extends User, Developer {
13  projects: string[];
14}
15
16const user: TeamLead = {
17  name: 'Daria',
18  isAwesome: true,
19  department: 'frontend',
20  projects: ['Awesome projects', 'Another awesome project']
21}

В отличии от автоматического слияния, расширение с помошью extends — это явное наследование одного интерфейса от другого.

Интерфейсы для классов

Класс может реализовать интерфейс. С помощью ключевого слова implements TypeScript заставляет класс строго соответствовать определенному интерфейсу:

1interface Animal {
2  name: string;
3  makeSound(): void;
4}
5
6class Dog implements Animal {
7  name: string;
8
9  constructor(name: string) {
10    this.name = name;
11  }
12
13  makeSound() {
14    console.log('Гав!');
15  }
16}

Если чего-то не хватит — TypeScript покажет ошибку:

1interface Animal {
2  name: string;
3  makeSound(): void;
4}
5
6class Dog implements Animal { // Class 'Dog' incorrectly implements interface 'Animal', Property 'makeSound' is missing in type 'Dog' but required in type 'Animal'
7  name: string;
8
9  constructor(name: string) {
10    this.name = name;
11  }
12}

В отличии от extends, который наследует код (свойства, методы, конструктор),implements только проверяет соответствие формы.

Ограничения интерфейсов

  • Нельзя использовать объединение (Union):

    1interface User {
2  name: string;
3}
4
5interface User {
6  isAwesome: boolean;
7}
8
9const user: User = {
10  name: "dtsiki",
11  isAwesome: true,
12};

  • С помощью интерфейсов нельзя описывать примитивы:

    1interface ID = string | number; // Error!

Литералы (Literal Types)

В TypeScript можно использовать не только общие типы, но и конкретные значения как типы. Литерал — это тип, который принимает ровно одно конкретное значение.

Расширение типов(Type Widening)

TypeScript по разному реагирует на переменные, объявленние с помощью ключевых слов let и const:

literal-types-example(1).ts
1const admin = "admin";
2let user = 'user';

В этом примере типом переменной admin будет не string, а строка admin. А вот типом переменной user будет как раз string. TypeScript превращает переменную admin в литерал.

Если вы объявите переменную с помощью const, TypeScript автоматически сузит её тип до литерального, так как её значение не может измениться.

Заморозка литералов(Const Assertions)

Можно добавить as const и присвоить литеральные типы:

literal-types-example(2).ts
1let user = 'user' as const;

Теперь user будет литералом с типом user.

Обычно это используется для заморозки объектов: все свойства автоматически помечаются как readonly, а их значения получают литеральные типы, чтобы избежать случайных ошибок перезаписи:

literal-types-example(3).ts
1const colorTheme = {
2  background: 'dark',
3  text: 'light'
4} as const;
5
6colorTheme.background = 'light'; // Ошибка: Cannot assign to 'background' because it is a read-only property

Виды литералов

В TypeScript существует 3 основных вида литеральных типов: строковые, числовые и логические.

  • строковые литералы

    literal-types-example(2).ts
    1type Status = 'idle' | 'loading' | 'success' | 'error'; // String literal type
2
3function handleStatus(status: Status) {
4  // статусом может быть только одним из четырёх значений
5}

  • числовые литералы

    literal-types-example(3).ts
    1type Dice = 1 | 2 | 3 | 4 | 5 | 6; // Number literal type
2
3function rollDice(): Dice {
4  return Math.floor(Math.random() * 6) + 1 as Dice;
5}

  • логические литералы

    literal-types-example(4).ts
    1type SuccessResult = {
2  success: true; // Boolean literal type
3  data: string;
4};
5
6type FailureResult = {

3  password: 'admin',
4} as const;

Перечисления или литеральные типы

Перечисления enum — это ещё способ задать набор именованных констант:

literal-types-example(7).ts
1enum Status {
2  IDLE = 'idle',
3  LOADING = 'loading',
4  SUCCESS = 'success',
5  ERROR = 'error',
6}

Теперь можно использовать Status.IDLE, Status.LOADING и т.д. — и получать соответствующие строки.

Рекомендуется избегать использования enum для именнованных констант и отдавать предпочтение типам и литералам. Связано это с тем, что типы и литералы — это чистый TypeScript, который исчезает после компиляции, в том время как перечисления — это реальный JavaScript-объект, который остаётся в рантайме и генерирует лишний код.

Например, после компиляции перечисление из примера выше будет выглядеть так:

index.js
1var Status;
2(function (Status) {
3    Status["IDLE"] = "idle";
4    Status["LOADING"] = "loading";
5    Status["SUCCESS"] = "success";
6    Status["ERROR"] = "error";
7})(Status || (Status = {}));

Тип же исчезнет при компиляции и не попадёт в бандл:

index.js
1// ничего нет

Как заменить перечисление на тип

Например, в вашем проекте сложилось использование перечислений в таком виде:

literal-types-example(8).ts
1enum Status {
2  IDLE = 'idle',
3  LOADING = 'loading',
4  SUCCESS = 'success',
5  ERROR = 'error'
6}
7
8function handleStatus(status: Status) {
9  ...
10}
11
12handleStatus(Status.IDLE);

Задача: убрать перечисления, но очень не хочется переходить на использование строк:

literal-types-example(9).ts
1type Status = 'idle' | 'loading' | 'success' | 'error';
2
3handleStatus('success');

Эта замена — один из самых частых рефакторингов в современном TypeScript. Для решения создадим объект с as const и затем выведем из него тип:

literal-types-example(10).ts
1const Status = {
2  IDLE: 'idle',
3  LOADING: 'loading',
4  SUCCESS: 'success',
5  ERROR = 'error'
6} as const;
7
8type Status = (typeof Status)[keyof typeof Status]; // 'idle' | 'loading' | 'success' | 'error'
9
10handleStatus(Status.IDLE);

Шаблонные литералы(Template Literal Types)

Позволяет строить типы на основе строковых шаблонов. Это как шаблонные строки в JavaScript, но на уровне типов.

1type World = "world";
2type Greeting = `hello ${World}`; // Тип: "hello world"
3
4// Комбинирование union типов
5type Direction = "top" | "bottom";
6type Margin = `margin-${Direction}`; // Тип: "margin-top" | "margin-bottom"

При использовании нескольких объединений шаблонный тип автоматически создаст все возможные комбинации:

1type Size = "small" | "large";
2type Color = "red" | "blue";
3type Style = `${Size}-${Color}`;
4// Тип: "small-red" | "small-blue" | "large-red" | "large-blue"

Шаблонные литералы можно использовать для типизации:

  • CSS-свойств:

    1type CssSize = `${number}${'px' | 'rem' | 'em'}`;
2
3const size: CssSize = '16px';
4const size2: CssSize = '2rem';
5// const invalid: CssSize = '16pt'; // Ошибка

  • экшенов:

    1type ActionType = 'add' | 'remove' | 'update';
2
3type Action<T> = {
4  type: `${ActionType}_${T}`;
5  payload: T;
6};
7
8// Создаём конкретный тип
9type UserAction = Action<'user'>;
10// { type: 'add_user' | 'remove_user' | 'update_user'; payload: 'user' }
11
12const action: UserAction = {
13  type: 'add_user',
14  payload: 'user',
15};

  • API-путей:

    1type HttpMethod = 'GET' | 'POST' | 'PUT' | 'DELETE';
2type ApiEndpoint = `/${string}`;
3
4const route: ApiEndpoint = '/users';
5// const invalid: ApiEndpoint = 'users'; // Ошибка

TypeScript предоставляет специальные функции для изменения регистра букв на уровне типов:

  • Uppercase<T> приводит к верхнему регистру

  • Lowercase<T> приводит к нижнему регистру

  • Capitalize<T> делает первую букву заглавной

  • Uncapitalize<T> делает первую букву строчной

Например:

1type EventName<T extends string> = `on${Capitalize<T>}`;
2type ClickEvent = EventName<"click">; // Тип: "onClick"

TypeScript позволяет динамически извлекать части строк или проверять их соответствие шаблону внутри условных типов c помощью ключевого слова infer:

1type GetterName<T extends string> = T extends `get${infer U}` ? U : never;
2type Prop = GetterName<"getUser">; // Тип: "User"

Дженерики (Generics)

Это инструмент, который позволяет создавать компоненты (функции, классы, интерфейсы), способные работать с различными типами данных без потери строгой типизации. Вместо жесткого указания типа, он передается как параметр.

Обобщённые функции

Позволяют зафиксировать тип аргумента так, чтобы функция возвращала значение того же типа.

Например есть функция, которая возвращает то, что получила. Без дженериков придётся использовать any:

1function print(value: any): any {
2  return value;
3}
4
5let str = print('Привет!'); // тип any
6let num = print(42); // тип any
7let isAwesome = print(true); // тип any

Из-за использования any тип переменной будет утерян.

Эту же функцию можно переписать с помощью дженерика, а тип переменной при этом будет сохранён:

1function print<T>(value: T): T {
2  return value;
3}
4
5let str = print('Привет!'); // тип string
6let num = print(42); // тип number
7let isAwesome = print(true); // тип boolean

Что здесь происходит:

  • С помощью <T> объявляем параметр типа. Можно назвать как угодно: Value, Something, но принято T или Type.

  • value: T — параметр функции будет иметь указанный перед этим.

  • : T — функция возвращает этот же тип

С помощью дженериков можно не привязываться к конкретно одному типу, а при вызове TypeScript автоматически будет подставлять конкретный тип.

В TypeScript можно использовать несколько параметров дженерика. Для этого их нужно перечислить в угловых скобках < > через запятую. Для второго параметра принято использовать букву U:

1function pair<T, U>(a: T, b: U): [T, U] {
2  return [a, b];
3}
4
5let pairResult = pair(42, "Привет"); // тип [number, string]

Для третьего параметра принято использовать букву V (следующая по алфавиту за T и U).

Три параметра часто нужны в сложных структурах данных. Например, для хранения текущего значения, предыдущего значения и метаданных:

1interface HistoryState<T, U, V> {
2  current: T;
3  previous: U;
4  metadata: V;
5}
6
7const pageState: HistoryState<string, string | null, number> = {
8  current: "/home",
9  previous: null,
10  metadata: 200 // HTTP код
11};

Используются когда структура данных должна оставаться гибкой.

При использовании с интерфейсами они работают как параметры функций:

1interface ApiResponse<T> {
2  data: T;
3  status: number;
4  message: string;
5}
6
7// Использование с конкретными типами
8const userResponse: ApiResponse<{ name: string; age: number }> = {
9  data: { name: 'Иван', age: 28 },
10  status: 200,
11  message: 'Success',
12};
13
14const stringResponse: ApiResponse<string[]> = {
15  data: ['apple', 'banana'],
16  status: 200,
17  message: 'Success',
18};

Объявления типов работают с дженериками точно так же, как и интерфейсы:

1type ApiResponse<T> = {
2  data: T;
3  status: number;
4  message: string;
5}
6
7const userResponse: ApiResponse<{ name: string; age: number }> = {
8  data: { name: 'Иван', age: 28 },

12

Ограничения(Constraints)

Это инструмент, который заставляет дженерики соответствовать определенной структуре. Они сужают "вседозволенность" обычных дженериков, позволяя компилятору гарантировать наличие нужных методов и свойств внутри функции или класса. Делается с помощью ключевого слова extends.

Например, без ограничения получим ошибку:

1function logLength<T>(value: T) {
2  console.log(value.length) // Ошибка: Property 'length' does not exist on type 'T'
3}

Если сузить круг возможных типов, передаваемых в дженерик, ошибки больше не будет:

1type Lengthwise = {
2  length: number;
3}
4
5function logLength<T extends Lengthwise>(item: T): void {
6  console.log(item.length);
7}
8
9logLength("Текст"); // Разрешено: у строки есть .length
10logLength([1, 2, 3]); // Разрешено: у массива есть .length
11logLength(42); // Ошибка: у числа нет .length

Обобщённые классы

Дженерики в классах позволяют создавать шаблоны, способные работать с различными типами данных без дублирования кода.

Синтаксис здесь такой же: в угловых скобках < > передаётся тип при создании экземпляра и затем TypeScript сам подставит нужный тип:

1class List<T> {
2  private items: T[] = [];
3
4  add(item: T): void {
5    this.items.push(item);
6  }
7
8  getItems(): T[] {
9    return this.items;
10  }
11}
12
13// Класс строго работает со строками
14const stringList = new List<string>();
15stringList.add("Яблоко");
16// stringList.add(5); // Ошибка компиляции
17
18// Класс строго работает с числами
19const numberList = new List<number>();
20numberList.add(10);

1type Box<T = string> = {

Условные типы (Conditional Types)

Позволяют создавать типы, которые меняются в зависимости от условий. Это if/else, но для типов:

1T extends U ? X : Y

Например:

1type IsString<T> = T extends string ? true : false;

Как использовать этот тип:

1type A = IsString<string>; // true
2type B = IsString<number>; // false

Что можно сделать с помощью условных типов:

  • Делать проверку на конкретный тип (см. пример выше)

  • Комбинировать с дженериками для возврата разных типов данных в зависимости от типа входных данных:

    1type Wrapped<T> = T extends string ? `string: ${T}` : T extends number ? `number: ${T}` : T;
2
3type A = Wrapped<"Привет">;   // "string: Привет"
4type B = Wrapped<42>;        // "number: 42"
5type C = Wrapped<boolean>;   // boolean

  • Фильтровать типы:

    1type OnlyStrings<T> = T extends string ? T : never;
2
3type A = OnlyStrings<"a" | "b" | 1 | 2>;   // "a" | "b"

    Тип never в объединениях ведет себя как пустое множество: он просто исчезает из объединения, никак не влияя на итоговый тип.

infer

Ключевое слово infer используется в TypeScript внутри условных типов для того, чтобы:

  • Вывести и сохранить в переменную часть другого типа:

    1type ReturnType<T> = T extends (...args: any[]) => infer R ? R : any;
2
3type MyFunc = () => string;
4type T1 = ReturnType<MyFunc>; // Тип T1 будет string

  • Узнать тип элемента, который лежит внутри массива: тип:

    1type ElementType<T> = T extends (infer E)[] ? E : never;
2
3type NumbersArray = number[];
4type T2 = ElementType<NumbersArray>; // Тип T2 будет number

  • Получить аргументы функции

    1type MyParams<T> = T extends (...args: infer P) => any ? P : never;
2
3function doSomething(abc: string, num: number, flag: boolean) {
4  return { id: 1, abc, num, flag };
5}
6
7type FuncArgs = MyParams<typeof doSomething>; // [abc: string, num: number, flag: boolean]

Сужение типов (Type Narrowing)

Позволяет ограничить общий тип до более конкретного на основе проверок в коде.

Например, объединения string | number можно уточнить к более узкому типу, например, string внутри блока кода:

1function print(value: string | number) {
2  value.toUpperCase(); // Error: Property 'toUpperCase' does not exist on type 'number'
3}

Это позволяет безопасно вызывать специфичные методы. В примере выше будет получена ошибка из-за того, что в случае, если будет передана переменная типа number, метод toUpperCase() не будет найден т.к. его просто не существует. Этот метод предназначен исключительно для работы со строками.

Функция, которая сужает тип, и будет называться type guard'ом.

С помощью typeof

Подойдет для примитивов:

1function process(value: string | number) {
2  if (typeof value === 'string') {
3    return value.toUpperCase();
4  }
5
6  return value.toFixed(2);
7}

С помощью instanceof

Подойдёт для классов:

1class Dog {
2  bark() {
3    console.log('Гав');
4  }
5}
6
7class Cat {
8  meow() {
9    console.log('Мяу');
10  }
11}
12
13function makeSound(animal: Dog | Cat) {
14  if (animal instanceof Dog) {
15    animal.bark();
16  } else {
17    animal.meow();
18  }
19}

С помощью оператора in

Можно проверять на наличие свойства:

1interface Car {
2  drive(): void;
3}
4
5interface Boat {
6  sail(): void;
7}
8
9function operate(vehicle: Car | Boat) {
10  if ('drive' in vehicle) {
11    vehicle.drive();
12  } else {
13    vehicle.sail();
14  }
15}

Custom Type Guard

Когда нужно что-то сложнее, то можно написать свою функцию-проверку:

1interface User {
2  name: string;
3  age: number;
4}
5
6function isUser(obj: any): obj is User {
7  return obj && typeof obj.name === 'string' && typeof obj.age === 'number';
8}
9
10function process(data: unknown) {
11  if (isUser(data)) {
12    // теперь TypeScript знает, что это User у него есть поле name
13    console.log(data.name);
14  }
15}

obj is User здесь предикат типа (type predicate). Это специальный синтаксис, который сообщает компилятору, что если функция возвращает true, то переданный аргумент фактически принадлежит к указанному типу.

Проверка на существование(Truthiness Narrowing)

Это проверка, которая автоматически сужает тип переменной внутри условных конструкций, основываясь на том, является ли значение «истинным» (truthy) или «ложным» (falsy) в JavaScript:

1function processName(name: string | null | undefined) {
2  if (name) {
3    console.log(name.toUpperCase());
4  }
5}

Значения null и undefined здесь отфильтровались автоматически во время проверки значения name. Таким образом можно отсечь null, undefined, "", 0, false

Сужение на основе равенства(Equality narrowing)

В этом случае сужение будет происходить на основе проверок равенства с помощью операторов ===, !==, == или !=:

1function doSomething(x: string | number, y: string) {
2  if (x === y) {
3    // x теперь string
4  }
5}

Дискриминирующие объединения(Discriminated Unions)

Это набор типов, у которых есть общее поле-дискриминатор. Обычно это type, kind или status. По этому полю TypeScript понимает какой именно тип будет нужен.

1// Общий тип "фигура" с полем-дискриминатором kind
2interface Circle {
3  type: 'circle';
4  radius: number;
5}
6
7interface Square {
8  type: 'square';
9  side: number;
10}
11
12interface Rectangle {
13  type: 'rectangle';
14  width: number;
15  height: number;
16}
17
18type Shape = Circle | Square | Rectangle;

Теперь TypeScript знает: если shape.type === 'circle', то нужным типом точно будет Circle:

1function assertNever(type: never): never {
2  throw new Error(`Неизвестная фигура: ${type}`);
3}
4
5function getArea(shape: Shape): number {
6  switch (shape.type) {
7    case 'circle':
8      // TypeScript знает: здесь shape — Circle
9      return Math.PI * shape.radius ** 2;
10    case 'square':
11      // TypeScript знает: здесь shape — Square
12      return shape.side ** 2;
13    case 'rectangle':
14      // TypeScript знает: здесь shape — Rectangle
15      return shape.width * shape.height;
16    default:
17      // Если добавим новую фигуру — попадём сюда
18      return assertNever(shape);
19  }
20}
21
22getArea({ type: 'circle', radius: 42 });

Маппинг типы(Mapped Types)

Позволяют создавать новые типы на основе существующих, путём перебора ключей и последующей модификации их. По сути это функция map, но для типов: проходимся по каждому ключу и что-то делаем с ним.

Есть два синтаксических подхода к написанию маппинга типов:

  • Применяем маппинг сразу к конкретному типу:

    1type User {
2  id: number;
3  name: string;
4  age?: number;
5}
6
7type NewType = {
8  [K in keyof User]: User[K];
9};

  • Используем дженерик:

    1type NewType<T> = {
2  [K in keyof T]: T[K];
3};

В обоих случаях можно использовать модификаторы ? (сделает свойство необязательным) и readonly (сделает свойство доступным только для чтения), а также встроенные типы. Во время создания нового типа можно добавлять или удалять свойства, используя префиксы + (по умолчанию) и -.

Например, с помощью маппинга типа сделаем все поля объекта необязательнымы:

1type User = {
2  name: string;
3  age: number;
4};
5
6type OptionalUser = {
7  [K in keyof User]?: User[K];
8};

В результате получится такой тип:

1type OptionalUser = {
2  name?: string | undefined;
3  age?: number | undefined;
4}

Можно сделать все поля доступными только для чтения:

1type User = {
2  id: number;
3  name: string;
4  age?: number;
5};
6
7type ReadOnly<T> = {
8  readonly [K in keyof T]: T[K];
9};
10
11const user: ReadOnly<User> = {
12  id: 42,
13  name: "веышлш"
14};
15
16user.name = 'dtsiki'; // Ошибка: Cannot assign to 'name' because it is a read-only property

Также доступна фильтрация через ключевое слово as:

1type StringKeys<T> = {
2  [P in keyof T as P extends string ? P : never]: T[P];
3};
4
5type User {
6  id: number;
7  name: string;
8  age: number;
9  email: string;
10  isAwesome: boolean;
11}
12
13type OnlyStrings = StringKeys<User>;

TypeScript уже содержит множество готовых встроенных типов (Utility Types) на базе Mapped Types:Pick<T, K>, Record<K, T>, Omit<T, K> и т.д. Всё это дженерик-версии маппинг-типов. Именно поэтому они работают с любым типом. Например:

1// Встроенный тип:
2type PartialUser = Partial<User>;
3
4// Под капотом маппинг тип:
5type Partial<T> = {
6  [K in keyof T]?: T[K];
7};

Рекурсивные типы(Recursive Types)

Типы, которые ссылаются сами на себя. Рекурсивные типы часто встречаются при работе с API, файловой системой, комментариями и древовидными структурами.

Ими можно типизировать:

  • Деревья:

    1type TreeNode = {

23};
24

  • JSON-объекты:

2  | string
3  | number
4  | boolean

Встроенные типы(Utility Types)

Встроенные типы помогают создавать новые на основе уже существующих.

Partial<Type>, Required<Type>, Readonly<Type>, Pick<Type, Keys>, Omit<Type, Keys>

Эти встроенные типы изменяют модификаторы существующих полей в объектах: делают их опциональными, обязательными или доступными только для чтения, а также позволяют фильтровать, выбирая или удаляя нужные поля из типов.

Важно: встроенные типы создают новый тип, а исходный при этом не меняется.

Для примера возьмём тип User:

1type User {
2  id: number;
3  name: string;
4  email: string;
5  age?: number;
6}

На его основе создадим новые с помощью встроенных типов:

  • Partial<Type> делает все свойства типа Type необязательными (опциональными):

    1type PartialUser = Partial<User>;
2
3const user1: User = {}; // Type '{}' is missing the following properties from type 'User': id, name, email
4const user2: PartialUser = {};

  • Required<Type> делает все свойства типа Type обязательными, даже если они были объявлены как опциональные:

    1type RequiredUser = Required<User>;
2
3const user: RequiredUser = { // Property 'age' is missing in type '{ id: number; name: string; email: string; }' but required in type 'Required<User>'
4  id: 42,
5  name: 'dtsiki',
6  email: 'dtsiki@dtsiki.dtsiki'
7};

  • Readonly<Type> делает все свойства типа Type доступными только для чтения, предотвращая последующие изменения:

    1type ReadonlyUser = Readonly<User>;
2
3const user: ReadonlyUser = {
4  id: 42,
5  name: 'dtsiki',
6  email: 'dtsiki@dtsiki'
7};
8
9user.email = 'dtsiki@dtsiki.dtsiki'; // Cannot assign to 'email' because it is a read-only property

  • Pick<Type, Keys> выбирает только указанные свойства Keys из типа Type:

    1type PublicUser = Pick<User, 'id' | 'name'>;
2
3const user: PublicUser = {
4  id: 42,
5  name: 'dtsiki',
6};

  • Omit<Type, Keys> удаляет указанные свойства Keys из типа Type:

    1type PublicUser = Omit<User, 'email' | 'age'>;
2
3const user: PublicUser = {
4  id: 42,
5  name: 'dtsiki',
6};

Всё выше перечисленное будет работать и для интерфейсов т.к. для TypeScript типы и интерфейсы это одно и то же — объектный тип:

1interface User {
2  id: number;
3  name: string;
4  email: string;
5  age?: number;
6}
7
8type PartialUser = Partial<User>;
9type RequiredUser = Required<User>;
10type ReadonlyUser = Readonly<User>;
11type PublicPickUser = Pick<User, 'id' | 'name'>;
12type PublicOmitUser = Omit<User, 'email' | 'age'>;

Record<Keys, Type>

Создает новый объект, где ключи имеют тип Keys, а значения — тип Тype.

Чаще всего используется для создания словарей (хеш-таблиц), где ключом является строка, а значением — любой нужный тип:

1type PetInfo = {
2  age: number;
3  breed: string;
4}
5
6const myPets: Record<string, PetInfo> = {
7  "Kesha": { age: 5, breed: "Pembroke Welsh Corgi" },
8  "Lucky": { age: 3, breed: "Mixed Breed Cat" },
9  "Smoke": { age: 16, breed: "Mixed Breed Cat" },
10};

Exclude<Type, U>, Extract<Type, U>, NonNullable<Type>

Позволяют фильтровать элементы в объединениях, например, оставлять только нужные.

  • Exclude<T, U> исключает из объединения T все типы, которые можно присвоить U:

    1type All = 'a' | 'b' | 'c' | 'd';
2type WithoutBAndC = Exclude<All, 'b' | 'c'>; // 'a' | 'd'

  • Extract<T, U> наоборот — оставляет только совпадающие:

    1type All = 'a' | 'b' | 'c' | 'd';
2type WithBandC = Extract<All, 'b' | 'c'>; // 'b' | 'c'

  • NonNullable<T> убирает из T все null и undefined:

    1type SomeStrangeType = string | null | undefined;
2type SomeDefiniteType = NonNullable<SomeStrangeType>; // string

Parameters<Type>, ReturnType<Type>, ConstructorParameters<Type>, InstanceType<Type>

Служат для извлечения информации о параметрах или возвращаемых значениях из сигнатур функций.

  • Parameters<Type> позволяет получить типы аргументов функции Type в виде кортежа:

    1function sayHello(name: string, age?: number): string {
2  return `Привет, ${name}`;
3}
4
5type Params = Parameters<typeof sayHello>; // [name: string, age?: number]

  • ReturnType<Type> позволяет получить тип возвращаемого значения из функции:

    1function fetchUser() {
2  return { id: 42, name: '@dtsiki', isAwesome: true };
3}
4
5type User = ReturnType<typeof fetchUser>; // { id: number; name: string; isAwesome: boolean; }

  • ConstructorParameters<Type> позволяет извлечь типы параметров конструктора класса в виде кортежа:

    1class User {
2  constructor(public name: string, public age: number) {}
3}
4
5type UserParams = ConstructorParameters<typeof User>; // [name: string, age: number]
6
7// Теперь можно использовать этот тип для функций-фабрик или оберток
8function createUser(...args: UserParams) {
9  return new User(...args);
10}

  • InstanceType<Type> позволяет извлечь тип, который возвращает конструктор класса:

    1class User {
2  id: number;
3  name: string;
4  isAwesome: boolean;
5}
6
7type UserInstance = InstanceType<typeof User>; // User

Awaited<Type>

Awaited<T> — специальная утилиты для работы с промисами, которая разворачивает (распаковывает) тип Promise, возвращая тип, который находится внутри него:

Самый частый сценарий использования — получение типа возвращаемого значения из асинхронных функций в сочетании с ReturnType:

1async function fetchUser(): Promise<{ id: number; name: string; isAwesome: boolean }> {
2  return { id: 42, name: 'dtsiki', isAwesome: true };
3}
4
5// Обычный тип функции: () => Promise<{ id: number; name: string; isAwesome: boolean  }>
6type UserFunctionReturnType = ReturnType<typeof fetchUser>;
7
8// Распакованный тип: { id: number; name: string; isAwesome: boolean  }
9type User = Awaited<UserFunctionReturnType>;

Объектно-ориентированное программирование в TypeScript

TypeScript полностью поддерживает классическое объектно-ориентированное программирование, предлагая строгую типизацию, классы, интерфейсы, модификаторы доступа, абстрактные классы и удобную реализацию наследования через ключевое слово extends.

Классы

В TypeScript есть полноценная поддержка классов, унаследованная из стандарта ECMAScript, с добавлением мощных возможностей строгой типизации.

1class User {
2  name: string;
3
4  constructor(name: string) {
5    this.name = name;
6  }
7
8  greet(): string {
9    return `Привет, ${this.name}!`;
10  }
11}
12
13const user = new User('dtsiki');
14console.log(user.greet()); // Привет, dtsiki!

Модификаторы доступа

В TypeScript есть четыре модификатора, которые управляют видимостью свойств и методов класса, обеспечивая инкапсуляцию:

  • public: свойства и методы с этим модификатором доступны из любого места — внутри класса, в дочерних классах и снаружи (в экземпляре объекта). Если модификатор не указан, по умолчанию свойство становится public.

  • private: свойства и методы доступны только внутри того класса, где они объявлены. Доступ к ним извне или из дочерних классов приведет к ошибке компиляции.

  • protected: свойства и методы доступны внутри самого класса, а также во всех его дочерних классах. Снаружи к ним обращаться нельзя.

  • readonly: свойства и методы доступы только для чтения, изменить их после создания или инициализации нельзя.

1class User {
2  public name: string; // доступно везде
3  private age: number; // только внутри класса
4  protected email: string; // внутри класса и наследников
5  readonly id: number; // только для чтения
6
7  constructor(id: number, name: string, age: number, email: string) {
8    this.id = id;
9    this.name = name;
10    this.age = age;
11    this.email = email;
12  }
13
14  private getAge(): number {
15    return this.age; // доступно внутри класса
16  }
17
18  public getInfo(): string {
19    return `${this.name} (${this.getAge()})`;
20  }
21}
22
23const user = new User(42, 'dtsiki', 33, 'dtsiki@dtsiki.dtsiki');
24
25// public:
26console.log(user.name); // dtsiki
27
28// private:
29console.log(user.age); // Ошибка: Property 'age' is private and only accessible within class 'User'
30
31// protected:
32console.log(user.email); // Ошибка: Property 'email' is protected and only accessible within class 'User' and its subclasses
33
34// readonly:
35user.id = 1; // Ошибка: Cannot assign to 'id' because it is a read-only property

TypeScript позволяет не объявлять свойства и не присваивать им значения вручную, а указать модификатор прямо в аргументах конструктора:

1class User {
2  constructor(readonly id: number, public name: string, private age: number, protected email: string) {
3    this.id = id;
4    this.name = name;
5    this.age = age;
6    this.email = email;
7  }
8  ...
9}

Наследование

Дочерний класс наследует свойства и методы родительского, но может добавлять свои или переопределять существующие. Наследование осуществляется с помощью ключевого слова extends, а в дочернем классе обязательно нужно вызывать метод super():

1class Animal {
2  constructor(public name: string) {}
3
4  makeSound(): void {
5    console.log(`${this.name} издает звук`);
6  }
7}
8
9class Dog extends Animal {
10  constructor(name: string, public breed: string) {
11    super(name); // Обязательно вызываем super для инициализации родительского конструктора
12  }
13
14  // Переопределение метода
15  makeSound(): void {
16    console.log(`${this.name} лает: Гав-гав!`);
17  }
18
19  fetch(): void {
20    console.log(`${this.name} приносит палку`);
21  }
22}
23
24const myDog = new Dog("Кеша", "корги");
25myDog.makeSound(); // Выведет: Кеша лает: Гав-гав!

В TypeScript нет множественного наследования классов, но его поддерживают интерфейсы — можно унаследовать сразу несколько интерфейсов через запятую с помощью ключевого слова implements (см. Реализация интерфейсов).

Абстрактные классы

Абстрактные классы нельзя инстанциировать. Они нужны как шаблон для других классов.

1abstract class AbstractUser {
2  constructor(public id: number, public email: string) {}
3
4  // Абстрактный метод — у него нет тела, дочерние классы обязаны его реализовать
5  abstract getRole(): string;
6
7  displayInfo(): void {
8    console.log(`Email: ${this.email}`);
9  }
10}
11
12class Admin extends AbstractUser {
13  getRole(): string {
14    return "Администратор";
15  }
16}
17
18const admin = new Admin(1, "admin@site.ru");
19admin.displayInfo(); // Email: admin@site.ru

Реализация интерфейсов

Ключевое слово implements в TypeScript заставляет класс строго соответствовать контракту, описанному в интерфейсе. Класс обязан реализовать все указанные свойства и методы; в противном случае компилятор выдаст ошибку:

1interface Animal {
2  name: string;
3  makeSound(): void;
4}
5
6interface Pet {
7  play(): void;
8}
9
10class Dog implements Animal, Pet {
11  constructor(public name: string) {}
12
13  makeSound() {
14    console.log(`${this.name} лает: Гав-гав!`);
15  }
16
17  play() {
18    console.log(`${this.name} приносит палку...`);
19  }
20}

В это пример,если не будет реализован, например, метод play(), то будет получена ошибка компиляции, сообщающая о том, что необходимо реализовать этот этот метод потому что он является обязательным в интерфейсе Pet.

Разница между extends и implements:

  • extends используют для наследования: класс берёт и копирует готовую реализацию от родительского класса.

  • implements: класс лишь обязуется реализовать все свойства и методы, описанные в интерфейсе.

Статические свойства и методы

В TypeScript нет выделенного ключевого слова для создания самих статических классов. Ключевое слово static помечает только свойства и методы, принадлежащие самому классу. Создать объект такого класса через невозможно.

1class MathUtils {
2  // Приватный конструктор запрещает создание экземпляров с помощью new
3  private constructor() {}
4
5  // Статическое свойство (константа)
6  static readonly PI = 3.14159;
7
8  // Статический метод
9  static calculateCircleArea(radius: number): number {
10    return this.PI * radius * radius;
11  }
12}
13
14// Использование напрямую, без new MathUtils()
15console.log(MathUtils.PI); // 3.14159
16console.log(MathUtils.calculateCircleArea(5)); // 78.53975

Статические классы используются для создания глобальных утилит (математические операции, работа со строками или датами), фабрик, сервисов и констант. Они группируют логику и данные, которые принадлежат самому классу, а не отдельным экземплярам, избавляя от необходимости вызывать new ClassName().

Геттеры и сеттеры

Геттеры и сеттеры — это специальные методы, которые позволяют управлять доступом к свойствам класса. Они полезны для инкапсуляции, выполнения проверок перед изменением данных или вычисления значений на лету.

Для их создания используются ключевые слова get и set, а при обращении к ним скобки не требуются:

1class User {
2  // Приватное поле, к которому нет прямого доступа извне
3  private _age: number;
4
5  constructor(age: number) {
6    this._age = age;
7  }
8
9  // Геттер: вызывается при чтении (user.age)
10  get age(): number {
11    return this._age;
12  }
13
14  // Сеттер: вызывается при записи (user.age = 25)
15  set age(newAge: number) {
16    if (newAge < 0) {
17      throw new Error("Возраст не может быть отрицательным");
18    }
19    this._age = newAge;
20  }
21}
22
23const user = new User(20);
24console.log(user.age); // 20 (сработал геттер)
25
26user.age = 25; // сработал сеттер
27// user.age = -5; // выбросит ошибку
28

Классы с дженериками

И тут не обойдёмся без них: дженерики позволяют создавать классы, которые могут работать с разными типами данных, сохраняя при этом строгую типизацию.

1class Stack<T> {
2  private items: T[] = [];
3
4  push(item: T): void {
5    this.items.push(item);
6  }
7
8  pop(): T | undefined {
9    return this.items.pop();
10  }
11
12  peek(): T | undefined {
13    return this.items[this.items.length - 1];
14  }
15}
16
17const numberStack = new Stack<number>();
18numberStack.push(1);
19numberStack.push(2);
20console.log(numberStack.pop()); // 2
21
22const stringStack = new Stack<string>();
23stringStack.push('a');
24stringStack.push('b');
25console.log(stringStack.pop()); // 'b'

Типизация DOM и событий

Для работы с DOM и событиями в TypeScript компилятор использует встроенные определения типов из файла lib.dom.d.ts. Там описаны Document, HTMLElement, HTMLInputElement, Event, MouseEvent и т.д.

Файл lib.dom.d.ts не нужно устанавливать вручную, он по молчанию подключается компилятором. Чтобы он работал, достаточно чтобы в вашем файле конфигурации TypeScript tsconfig.json был правильно указан массив "lib". Если вы используете среду браузера, в настройках должен быть включен параметр "dom". Например:

tsconfig.json
1{
2  "compilerOptions": {
3    "lib": ["es2022", "dom"]
4  }
5}

Проверка типов перед использованием

Методы вроде getElementById могут вернуть null, если элемента нет, поэтому TypeScript требует проверку:

1const awesomeButton = document.getElementById("awesome-button");
2
3if (awesomeButton) {
4  awesomeButton.addEventListener("click", () => {
5    console.log("Кликнули на кнопку");
6  });
7}

Приведение типов(Type Assertions)

Иногда TypeScript не может точно определить элемент, например, при поиске по классу. Можно явно указать тип с помощью конструкции as:

1const input = document.querySelector("input") as HTMLInputElement;

Важно понимать: это не преобразование, а лишь подсказка компилятору. При сборке в JavaScript эти конструкции будут удалены.

Type Guards

Альтренатива приведению типов с as — проверка класса объекта через instanceof:

1const element = document.getElementById("my-element");
2
3if (element instanceof HTMLImageElement) {
4  // Внутри этого блока element имеет тип HTMLImageElement
5  element.src = "image.png";
6}

Полезные типы DOM

Рано или поздно вам пригодится каждый из этих типов:

  • HTMLElement: базовый тип для всех HTML-элементов. От него наследуются все остальные типы HTML-элементов. Например:

    • HTMLInputElement: тип для тега <input>

  • Event: базовый тип для событий. От него наследуются все остальные события. Например:

    • MouseEvent: событие, описывающее любое действие пользователя мышью (клики, наведение, прокрутка)

    • KeyboardEvent: событие, возникающее при взаимодействии с клавиатурой (нажатие, удержание или отпускание клавиши)

Типизация React

TypeScript поддерживает большинство популярных веб-фреймворков, в том числе и React.

Написание кода на React в связке с TypeScript очень похоже на JavaScript/React код, ключевая разница будет заключаться только в том, что нужно будет указывать типы для компонентов, пропсов, стейтов и функций.

Настройка TypeScript

Чтобы использовать TypeScript в React-проектах достаточно использовать расширение .tsx для файлов компонентов и установить типы:

  • Для новых проектов лучший способ — инициализировать приложение через поддерживаемые фреймворки (например, с помощью Vite или Next.js), выбрав шаблон с TypeScript

  • Для ручного подключения установите типы командой:

    npm i -D typescript @types/react @types/react-dom

Типизация пропсов

Для типизации пропсов используйте ключевое слово type или interface. Создайте отдельный тип для пропсов и задайте его для функционального компонента:

Button.tsx
1interface IButtonProps = {
2  label: string;
3  onClick: () => void;
4  disabled?: boolean;
5};
6
7export const Button = ({ label, onClick, disabled = false }: IButtonProps) => {
8  return (
9    <button onClick={onClick} disabled={disabled}>
10      {label}
11    </button>
12  );
13};

Что выбрать: interface или type? Для пропсов они взаимозаменяемы, но часто выбирают interface.

Можно создать в директории компонента отдельный файл COMPONENT_NAME.types.ts и перенести тип/интерфейс для пропса туда — это отличная практика для очистки кода компонентов:

Button.types.ts
1export interface IButtonProps = {
2  label: string;
3  onClick: () => void;
4  disabled?: boolean;
5};
Button.tsx
1import { IButtonProps } from './types';
2
3export const Button = ({ label, onClick, disabled = false }: IButtonProps) => {
4  return (
5    <button onClick={onClick} disabled={disabled}>
6      {label}
7    </button>
8  );
9};

Типизация компонентов

Раньше надо было типизировать компоненты с FC (или FunctionComponent):

Button.types.tsx
1export interface IButtonProps = {
2  label: string;
3  onClick: () => void;
4};
Button.tsx
1import { FC } from 'react';
2import { IButtonProps } from './types';
3
4export const Button: FC<IButtonProps> = ({ label, onClick }) => {
5  return (
6    <button onClick={onClick}>
7      {label}
8    </button>
9  );
10};

Внимание: это старый подход!

В современном TypeScript это делать не обязательно и часто не рекомендуется — FC автоматически добавляет children: ReactNode в пропсы компонента, а также тип FC не принимает дженерики в пропсах. Это может привести к ситуациям, когда компонент, который не должен принимать дочерние элементы, всё равно их принимает без ошибок со стороны TypeScript. Поэтому достаточно просто протипизировать пропсы без типизации самого компонента:

Button.types.tsx
1export interface IButtonProps = {
2  label: string;
3  onClick: () => void;
4};
Button.tsx
1import { IButtonProps } from './types';
2
3export const Button = ({ label, onClick }:IButtonProps) => {
4  return (
5    <button onClick={onClick}>
6      {label}
7    </button>
8  );
9};

Типизация children

Если компонент должен оборачивать другие компоненты т.е. принимать children, используйте стандартный тип ReactNode:

Header.types.ts
1import { ReactNode } from 'react';
2
3export interface IHeaderProps = {
4  children: ReactNode;
5}
Header.tsx
1import { IHeaderProps } from './types';
2
3export const Header = ({ children }: IHeaderProps) => {
4  return <header>{children}</header>;
5};

Этот тип настоящая палочка-выручалочка и представляет собой всё, что может быть отрендерено в React: элементы React (JSX), строки, числа, массивы, фрагменты, null, undefined и логические значения.

Типизация событий

События типизируются встроенными типами из React. В зависимости от типа действия (клик, ввод, отправка формы) используются специфические типы, такие как MouseEvent, ChangeEvent или FormEvent, соответственно.

События мыши(Mouse Events)

Для кликов, наведений и других манипуляций мышью используйте тип MouseEvent. В дженерике < > обычно указывается HTML-тег элемента, с которого приходит событие, в данном случае это кнопка поэтому тип будет HTMLButtonElement:

Button.ts
1import { MouseEvent } from 'react';
2
3  const handleClick = (event: MouseEvent<HTMLButtonElement>) => {
4  console.log('Кнопка нажата', event.currentTarget);
5}
6
7const Button = () => {
8  return <button onClick={handleClick}>Нажми меня</button>;
9};

События ввода(Change Events)

Если нужно обработать изменение в поле ввода, селекторе или чекбоксе, используйте ChangeEvent. В качестве дженерика нужно передать элемент формы:

Input.ts
1import { ChangeEvent } from 'react';
2
3const Input = () => {
4  const [value, setValue] = useState<string>('');
5
6  const handleChange = (event: ChangeEvent<HTMLInputElement>) => {
7    setValue(event.target.value);
8  };
9
10  return <input type="text" value={value} onChange={handleChange} />;
11};

Остальные события работают аналогично. Если не понятно, какой тип следует присвоить, то в React с TypeScript для определения типа события достаточно: либо воспользоваться подсказкой вашей среды разработки, наведя курсор на место, где будет передан аргумент события.

  • Посмотреть на атрибут тега, например, onClick или onChange. Начните писать тег, напишите атрибут, наведите на слово event в IDE и TypeScript покажет, что это MouseEvent<HTMLButtonElement>:

    Button.tsx
    1<button onClick={(event) => {} />

  • Воспользоваться подсказкой вашей среды разработки, наведя курсор на место, где будет передан аргумент события. Похоже на предыдущий способ: вынесите функцию, но не пишите тип сразу, наведите на event, IDE снова подскажет нужный тип:

    Button.tsx
    1const handleClick = (event) => {
2  console.log(event.currentTarget);
3};

В большинстве проектов вам понадобятся лишь несколько основных типов. Главное правило: ChangeEvent используется, когда меняется значение (ввод), а MouseEvent для кликов или перемещений. В качестве дженерика нужно передать тип HTML-элемента.

Типизация хуков

Типизация хуков происходит автоматически, но часто требует явного указания типов при работе с начальными состояниями, сложными объектами или массивами.

useState

Типизация useState обычно выводится автоматически из начального значения. Тип можно указывать:

1const [label, setLabel] = useState<string>('Hello');
2const [num, setNum] = useState<number>(42);
3const [isAwesome, setIsAwesome] = useState<boolean>(true)

А можно и пропустить, как и в случае с переменными (см. Аннотация типов) — TypeScript сам может автоматически определить тип по начальному значению:

1const [label, setLabel] = useState('Hello'); // тип string
2const [num, setNum] = useState(42); // тип number
3const [isAwesome, setIsAwesome] = useState(true); // тип boolean

Если начальное значение может быть null или тип неочевиден, используйте дженерик в виде useState<Type>(initialValue) c объединением для явного указания:

1interface User {
2  id: number;
3  name: string;
4}
5
6const [user, setUser] = useState<User | null>(null);

Если начальное состояние это массив объектов, который изначально пуст:

1interface User {
2  id: number;
3  name: string;
4}
5
6const [users, setUsers] = useState<User[]>([]);

useRef

Хук useRef используется как для хранения значений, так и для ссылок на DOM-элементы, поэтому есть два способа его типизации, важно передавать точный тип элемента и начальное значение в зависимости от варианта использования хука:

  • useRef хранит ссылку на DOM-элемент:

    1const inputRef = useRef<HTMLInputElement>(null);

  • useRef используется как переменная для сохранения данных:

    1const timerId = useRef<number | null>(null);

Если нужно пробросить ref в компонент как пропс, то нужно обернуть дочерний компонент в forwardRef и проставить соответствующие типы:

1import { ReactNode } from 'react';
2
3export interface IButtonProps = {
4  children: ReactNode;
5  onClick: () => void;
6};
1import { IButtonProps } from './types'
2
3export const Button = forwardRef<HTMLButtonElement, IButtonProps>(
4  ({ onClick, children }, ref: ForwardedRef<HTMLButtonElement>) => {
5    return (
6      <button ref={ref} onClick={onClick}>
7        {children}
8      </button>
9    );
10  }
11);
12
13Button.displayName = 'Button';

Кастомные хуки

Для создания кастомных хуков применяются те же правила типизации аргументов функций и возвращаемых значений. Возвращаемый результат часто типизируется как кортеж:

1function useCounter(initialValue: number = 0): [number, () => void, () => void] {
2  const [count, setCount] = React.useState(initialValue);
3
4  const increment = () => setCount(c => c + 1);
5  const decrement = () => setCount(c => c - 1);
6
7  return [count, increment, decrement];
8}

Если хук возвращает более 2-3 значений, лучше вернуть объект. Это сделает код при деструктуризации независимым от порядка следования переменных и самодокументируемым:

1type UseFormReturn<T> = {
2  values: T;
3  handleChange: (e: React.ChangeEvent<HTMLInputElement>) => void;
4  reset: () => void;
5};
6
7function useForm<T extends Record<string, any>>(initialValues: T): UseFormReturn<T> {
8  const [values, setValues] = React.useState(initialValues);
9
10  const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
11    setValues({ ...values, [e.target.name]: e.target.value });
12  };
13
14  const reset = () => setValues(initialValues);
15
16  return { values, handleChange, reset };
17}

Так же можно использовать дженерики для создания хуков.

useContext

Чтобы типизировать хук useContext, нужно задать интерфейс для данных контекста на этапе создания через createContext:

1interface ThemeContextType = {
2  theme: 'light' | 'dark';
3  toggleTheme: () => void;
4};
5
6const ThemeContext = createContext<ThemeContextType | null>(null);

Передаем null в качестве значения по умолчанию, чтобы избежать ложных срабатываний.

TypeScript автоматически подхватит указанный тип при вызове useContext. Для избежания проблем с null рекомендуется создавать кастомный хук-обертку, который будет проверять, что компонент находится внутри провайдера:

1export const useTheme = () => {
2  const context = useContext(ThemeContext);
3
4  if (!context) {
5    throw new Error('useTheme должен использоваться внутри ThemeProvider');
6  }
7
8  return context;
9};
1type ThemeProviderProps = {
2  children: ReactNode;
3};
4
5export const ThemeProvider = ({ children }: ThemeProviderProps) => {
6  const [theme, setTheme] = useState<'light' | 'dark'>('light');
7
8  const toggleTheme = () => {
9    setTheme((prev) => (prev === 'light' ? 'dark' : 'light'));
10  };
11
12  return (
13    <ThemeContext.Provider value={{ theme, toggleTheme }}>
14      {children}
15    </ThemeContext.Provider>
16  );
17};
1const ThemedButton = () => {
2  const { theme, toggleTheme } = useTheme(); // Типы подхватятся автоматически
3
4  return (
5    <button onClick={toggleTheme} style={{ background: theme === 'light' ? '#fff' : '#333' }}>
6      Сменить тему
7    </button>
8  );
9};

Компоненты-дженерики

С помощью дженериков можно делать универсальные компоненты, которые будут работать с разными типами данных, сохраняя при этом строгую типизацию.

Самый базовый пример такого компонента-дженерика это компонент-список:

List.types.ts
1import { ReactNode } from 'react';
2
3export interface IListProps<T> {
4  items: T[];
5  renderItem: (item: T) => ReactNode;
6}
List.tsx
1import { IListProps } from './types';
2
3export const List = <T,>({ items, renderItem }: IListProps<T>) => {
4  return (
5    <ul>
6      {items.map((item, index) => (
7        <li key={index}>{renderItem(item)}</li>
8      ))}
9    </ul>
10  );
11};

Дженерики-хуки

Дженерики также позволяют создавать переиспользуемые хуки, которые адаптируют свои типы под передаваемые данные. Например:

useLocalStorage.ts
1import { useState, useEffect } from 'react';
2
3function useLocalStorage<T>(key: string, initialValue: T): [T, (value: T | ((val: T) => T)) => void] {
4  const [storedValue, setStoredValue] = useState<T>(() => {
5    try {
6      const item = window.localStorage.getItem(key);
7      return item ? JSON.parse(item) : initialValue;
8    } catch (error) {
9      console.log(error);
10      return initialValue;
11    }
12  });
13
14  useEffect(() => {
15    try {
16      window.localStorage.setItem(key, JSON.stringify(storedValue));
17    } catch (error) {
18      console.log(error);
19    }
20  }, [key, storedValue]);
21
22  return [storedValue, setStoredValue];
23}

При вызове useLocalStorage<string>('user', 'dtsiki') TypeScript автоматически выведет все типы как string.
